서비스 전반에서 발생하는 공통 UI/UX 흐름·컴포넌트·정책을 누적 정리한 기획·개발 협업 문서

00
프롬프트 Prompt
다른 채팅창에서 이 파일을 동일하게 운영하기 위한 AI 운영 지침. 복사 후 새 채팅 첫 메시지로 붙여넣으세요.
AI 운영 프롬프트 · v1.9
파일 목적

안전NOW_v1_컴포넌트_정의서.html은 서비스 전반에서 발생하는 공통 UI/UX 흐름·컴포넌트·정책을 누적 정리하는 기획·개발 협업 문서다.
사용자가 UI/UX 가이드 내용을 입력하면 아래 규칙에 따라 이 파일에 반영하라.

파일 구조
섹션id용도수정 권한
00 s0 프롬프트 (운영 지침) ❌ 절대 금지
01 s1 업데이트 내역 행 추가만
02~ s2~ UI/UX 패턴 (순차 추가) 섹션 추가만
절대 규칙 — 위반 금지
  • 기존 섹션의 내용을 수정하거나 삭제하지 않는다.
  • <style> 내 기존 CSS를 삭제하거나 임의로 변경하지 않는다.
  • 사이드바의 기존 <a> 링크를 수정하거나 삭제하지 않는다.
  • 업데이트 내역은 최신순 5개까지 유지한다. 6번째 항목 추가 시 가장 오래된 행을 삭제한다.
  • 섹션 번호(00·01·02~)를 재정렬하거나 임의로 변경하지 않는다.
  • 기능정의서 바로가기 플로팅 버튼(.frd-btn)을 수정하거나 삭제하지 않는다.

새 UI/UX 섹션 추가

위치: #changelog-table <tbody> 최상단 삽입 (최신순 유지) / 최대 5개 유지 — 6번째 추가 시 가장 오래된 행 삭제
4
사이드바 버전·날짜 갱신

이미지 기반 UX 입력 처리
1
화면 이미지 분석
첨부된 이미지에서 화면 상태(빈·진행·완료·오류)와 UX 흐름(상태 → 행동 → 피드백)을 단계별로 파악한다.
2
규칙 추출 — AS-IS / TO-BE 정리
사용자 명시 규칙("~없어도 됨", "~없이 처리", "~필요")을 기준으로 변경 전후를 정리한다. 명시되지 않은 부분은 현재 화면 기준으로 문서화한다.
3
섹션 구성 — 블록 조합
화면명 → 섹션 제목 / .demo 상태별 시뮬레이션 (.demo-lbl 반드시 "예시" 포함) · .spec-grid 제약 조건 · .tbl 단계별 흐름 · .dd DO/DON'T (AS-IS vs TO-BE)
4
섹션 번호 부여
기존 마지막 섹션 번호 + 1. 사이드바 링크·업데이트 내역·버전은 위의 업데이트 규칙에 따라 함께 반영한다.
